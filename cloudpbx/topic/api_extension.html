  • API Guide

    Integrate a third-party software to the PBX via API interfaces.

    • Change Logs of API Version

      When sending an API request, the request address must include the API version. All request samples in this guide use {api version} to represent the API version.

    • API Introduction

      Yeastar Cloud PBX provides API interfaces for you to integrate a third-party software or device.

    • API Interfaces

      References of API interfaces and API samples.

      • API Authentication
      • System
      • Extension
        • Query Extension List

          Get basic information of the PBX extension list, like extension number, extension name, extension status, type, etc.

        • Query Extension Settings

          Get detailed information about an individual extension or multiple extensions. For example, developers could query the advanced settings of an extension. When there are multiple requests, separate them with commas.

        • Modify an Extension

          Configure the settings of an individual extension, such as extension number, name, concurrent registrations, email address, mobility extension, etc.

        • Query Voicemail Files

          Query voicemail files of a specific extension.

        • Download Voicemail Files

          Download voicemail files of a specific extension.

        • Delete Voicemail Files

          Delete voicemail files of a specific extension.

      • Trunk
      • IVR
      • Queue
      • Audio Calls
      • Play Prompts
      • Get Recording Files

        Through this interface, you can get auto recording files of the PBX.

      • Save and Apply Settings

        By default, when you configure the PBX settings via API interfaces, the settings will be saved and applied automatically. To avoid that the changed settings not being applied, you can invoke this interface to apply all settings after all settings are configured.

    • System Report

      The PBX will send report to the application server automatically when there is a change of system configuration or extension status, etc.

    • Call Report

      The PBX will send call report to the application server automatically. Call event like call transfer, call failure, call released, etc. will be reported.
