  • Admin Guide

    Admin Guide for the administrators who need to prepare for, configure and operate Yeastar Cloud PBX.

    • Extensions & Phones

      Configure extensions on the PBX web interface, manage voicemails of extensions. and register extensions on the IP phones or soft phones.

    • Trunk Management

      Set up trunks to make outbound calls or receive inbound calls.

    • Call Routes
      • Outbound Route

        An outbound route works like a traffic cop giving directions to road users to use a predefined route to reach a predefined destination.

        • Add an Outbound Route

          To allow users make outbound calls through trunks, you need to set up at least an outbound route on the PBX.

        • Adjust Priority of Outbound Routes

          The PBX will search for a matching dial pattern from the top route. If a route with higher priority is matched, the PBX will not continue going down the route list to search a "better" route.

        • Dial Patterns of Outbound Route

          To make an outbound calls from the PBX, users need to dial according to dial pattern settings of the outbound routes.

        • Outbound Route Examples

          Sample configurations that will help you understand outbound route dial patterns.

        • Restrict Outbound Calls

          The PBX has a default rule to limit how many outbound calls a user can make during a certain time period.

        • Add Emergency Numbers

          Emergency calls have the highest priority. Extension users can make emergency calls at any time.

      • Inbound Route

        An inbound route is used to tell the PBX where to route inbound calls based on the caller's phone number or the DID number. Inbound routes are often used in conjunction with time conditions and an IVR.

      • Time Conditions

        Time conditions are used to control call flow based upon time and date.

      • AutoCLIP Route

        Yeastar Cloud PBX can automatically stores records of outgoing call to AutoCLIP route table. When the called person calls back, the call will be routed directly to the original caller's extension.

      • SLA

        Shared Line Appearance (SLA) feature helps users share SIP trunks. It also helps monitor the status of the shared line. SLA feature works with BLF key on IP phones.

      • Adapt Caller ID

        Yeastar Cloud PBX supports adapting Caller ID for trunks. If the incoming Caller ID is adapted, you can press the call record directly on your phone to call back a number.

    • Call Features

    • Blacklist/Whitelist

      Whitelist is used to allow incoming/outgoing numbers. Blacklist is used to block an incoming/outgoing call.

    • Voice Prompts
    • System Management

    • Maintenance
    • PBX Monitor

      The PBX monitors the status of Trunks, Extensions, Concurrent Call, Conference.

    • Resource Monitor

      Monitor the CPU usage, memory usage, disk utilization and network flow.

    • CDR and Recordings

      You can check CDR and auto recordings on the PBX web interface. CDR (Call Detail Record) is a data record that contains various attributes of the call, such as time, duration, call status, source number, and destination number, etc.

Adjust Priority of Outbound Routes

The PBX will search for a matching dial pattern from the top route. If a route with higher priority is matched, the PBX will not continue going down the route list to search a "better" route.

The route priority is important, especially if there is some overlap. For example, the number 5503305 will match both a dial pattern of ZXXXXXX and X..
  1. Go to Settings > PBX > Call Control > Outbound Routes.
  2. Click the buttons to adjust the priority of your outbound routes.
